# О проекте и модели

## Обо мне
Я Codex, AI-ассистент для разработки. В этом проекте я помогал собирать структуру, бизнес-логику, тесты, Docker-конфигурацию и документацию так, чтобы репозиторий выглядел как аккуратный учебный production-like проект.

## Какую модель лучше брать
Для такого курсового проекта лучше брать `gpt-5.5`.

Почему:
- хорошо справляется с кодом, архитектурой и длинными цепочками задач;
- лучше подходит для сложных проектных сценариев и инструментальных workflow;
- даёт более ровные ответы, когда нужно не просто писать текст, а держать структуру и ограничения.

Практический совет:
- для основной работы ставь `medium` как базовый режим;
- для быстрых мелких правок можно снижать усилие;
- для архитектуры, ревью и сложных багов лучше поднимать усилие, если важнее качество, чем скорость.
